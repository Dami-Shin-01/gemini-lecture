## NotebookLM이란?

NotebookLM은 Google이 만든 AI 리서치 도구입니다. 문서를 업로드하면 해당 문서의 내용만을 기반으로 질문에 답하고, 요약하고, 연결 관계를 찾아줍니다.

## 심화: RAG는 왜 환각이 적은가

일반 LLM은 "학습된 지식"을 바탕으로 답을 만듭니다. 모르는 주제도 그럴듯하게 지어냅니다(환각). **RAG는 구조가 다릅니다.**

<ComparisonTable
  headers={["단계", "일반 LLM", "RAG (NotebookLM)"]}
  rows={[

    { label: "② 근거 탐색", cells: ["없음", "의미 기반(임베딩) 검색으로 상위 N개 구절 확보"] },
    { label: "③ 답변 생성", cells: ["학습 파라미터에서 직접", "검색된 구절을 컨텍스트로 넣고 생성"] },
    { label: "④ 답변 특성", cells: ["출처 불명·환각 가능", "출처 앵커 있음·환각 감소"] }
  ]}
/>

<HighlightBox type="expert" title="R&D 관점 한 줄 요약">
RAG = "검색 결과를 컨텍스트로 주입한 생성". 의미 기반 벡터 검색이라 키워드가 달라도 뜻이 맞으면 찾아냅니다 — 이게 비정형 현장 자료(간담회·이메일·설문)에 특히 강한 이유입니다.
</HighlightBox>

## 심화 1: 출처 추적 (Grounding) 실습

RAG의 핵심 능력은 "답변의 각 문장이 어느 소스의 어느 구절에 앵커되어 있는지" 추적 가능하다는 점입니다. 경영진 보고에서 **"데이터 근거"** 요청이 올 때 즉각 출처를 제시할 수 있어야 합니다.

## 심화 2: 감정 강도 축 재분류

기본 요약은 "얼마나 자주 언급됐는가" 축입니다. 의사결정 맥락에선 **"얼마나 강하게 느끼고 있는가"** 축이 더 중요할 때가 많습니다. 같은 노트북에 추가 질문을 던져 축을 바꿉니다.

<PromptBlock title="감정 강도 재분류 프롬프트">
{`위 답변의 이슈 TOP 3를 아래 기준으로 재정렬해줘.

[재정렬 기준]
- 감정 강도 = 분노(5) / 실망(4) / 우려(3) / 기대(2) / 중립(1)
- 강도 점수 × 언급 빈도 = 우선순위 점수

[출력 형식]
각 이슈마다:
1. 강도 점수 + 근거 인용문 1개 (소스명 포함)
2. 언급 빈도
3. 최종 우선순위 점수
4. 표 형태로 정리`}
</PromptBlock>

<HighlightBox type="tip" title="왜 이 질문이 의미 있나">
빈도순 정렬은 "가장 흔한 불만"을 보여줍니다. 강도×빈도 정렬은 "이탈·사기저하 리스크가 큰 이슈"를 보여줍니다. 경영진 의사결정 맥락에서 후자가 훨씬 실행력 있는 근거가 됩니다.
</HighlightBox>
