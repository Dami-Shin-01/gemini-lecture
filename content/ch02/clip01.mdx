# 대화 캔버스 프레임워크

프롬프트 엔지니어링이란 거창한 것이 아닙니다. **"AI에게 할 말을 잘 정리하는 기술"**입니다. 대화 캔버스 프레임워크(PTCF)는 이를 체계적으로 도와줍니다.

## 4가지 요소 — PTCF

### 1. 역할 (Persona)
AI에게 어떤 전문가로서 답변할지 알려줍니다.

### 2. 작업 (Task)
구체적으로 무엇을 해야 하는지 명확하게 요청합니다.

### 3. 맥락 (Context)
상황과 배경 정보를 제공합니다.

### 4. 형식 (Format)
결과물의 형태, 분량, 조건을 지정합니다.

<HighlightBox type="expert" title="프레임워크 핵심">
  모든 요소를 매번 사용할 필요는 없습니다. 
  **역할(P) + 작업(T) + 형식(F)**, 이 3가지만 잘 써도 결과가 크게 달라집니다.
</HighlightBox>

## 실전 예시

간단한 요청부터 시작해봅시다:

<PromptBlock title="기본 프롬프트">
{`분기 실적 보고서를 작성해줘`}
</PromptBlock>

이제 대화 캔버스(PTCF)를 적용해봅시다:

<PromptBlock title="대화 캔버스 적용">
{`[역할(P)] 너는 10년차 경영기획팀 매니저야.

[작업(T)] 경영진 보고용 분기 실적 요약 보고서를 작성해줘.

[맥락(C)] 2024년 3분기 실적을 경영진에게 보고해야 해. 
매출은 전분기 대비 12% 성장했고, 신규 고객이 340명 유입됐어.

[형식(F)] 
- A4 1장 분량
- 핵심 지표 → 분석 → 시사점 순서
- 불릿 포인트 활용
- 전문 용어는 최소화하고, 수치는 반드시 포함`}
</PromptBlock>

<HighlightBox type="tip" title="연습 포인트">
  위 두 프롬프트를 직접 Gemini에 입력해보세요. 
  결과물의 **구체성과 완성도** 차이를 직접 체감할 수 있습니다.
</HighlightBox>
