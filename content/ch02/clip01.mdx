# 간담회 기획안 & 진행 슬라이드 만들기

구성원의 목소리를 제대로 수집하려면, 간담회 자체가 잘 설계되어야 합니다. Gemini로 기획안과 진행 슬라이드 초안을 빠르게 만들어봅시다.

## 실습 1: 간담회 기획안 작성

<StepByStep
  steps={[
    {
      number: 1,
      title: "목적과 대상 정의",
      description: "간담회의 목적(VoE 수집? 고충 청취? 아이디어 발굴?)과 참석 대상(팀 전체? 특정 직급?)을 먼저 정합니다.",
    },
    {
      number: 2,
      title: "기획안 프롬프트 입력",
      description: "아래 프롬프트를 본인 상황에 맞게 수정하여 Gemini에 입력합니다.",
    },
    {
      number: 3,
      title: "결과 검토 & 개선",
      description: "생성된 기획안을 확인하고, '시간배분을 조정해줘', '아이스브레이킹 활동 추가해줘' 등 후속 요청으로 다듬습니다.",
    }
  ]}
/>

<PromptBlock title="간담회 기획안 프롬프트">
{`너는 구성원 소통 프로그램을 기획하는 전문가야.

[작업]
다음 조건으로 간담회 기획안을 작성해줘.

[맥락]
- 목적: 현장 구성원의 업무 고충과 개선 아이디어 청취 (VoE)
- 대상: 팀원 20~30명
- 시간: 2시간
- 형식: 소그룹 토론 + 전체 공유

[결과물 형식]
1. 간담회 목적 및 기대 효과 (2~3줄)
2. 진행 타임라인 (시간대별 활동)
3. 소그룹 토론 주제 3가지
4. 진행자 유의사항`}
</PromptBlock>

## 실습 2: 진행 슬라이드 구성

<PromptBlock title="슬라이드 구성 프롬프트">
{`위 간담회 기획안을 바탕으로 진행용 슬라이드 목차를 만들어줘.

- 슬라이드 수: 10장 이내
- 각 슬라이드에 제목 + 핵심 내용 2~3줄
- 참석자들이 보기 편한 구성으로
- 마지막 슬라이드에는 오늘 이야기를 마무리하는 메시지 포함`}
</PromptBlock>
