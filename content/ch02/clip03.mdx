# VoE 수집 결과 요약 & 분석하기

간담회가 끝나면 쏟아지는 의견들을 정리해야 합니다. 메모, 포스트잇, 회의록 등 날것의 VoE 데이터를 Gemini로 빠르게 분석해봅니다.

## 실습 1: 의견 요약 & 분류

간담회에서 수집한 의견들을 붙여넣고 패턴을 찾아봅니다.

<PromptBlock title="VoE 의견 분류 프롬프트">
{`너는 HR 전문가야. 아래 구성원 의견들을 분석해줘.

[작업]
1. 의견을 유형별로 분류 (업무 환경 / 소통 / 성장 / 복지 / 기타)
2. 가장 많이 언급된 주제 TOP 3 도출
3. 긴급하게 해결이 필요한 이슈 식별
4. 경영진에게 전달할 핵심 메시지 3줄 요약

[의견 데이터]
(여기에 간담회에서 수집한 의견들을 붙여넣으세요)
예시:
- 회의가 너무 많아서 실제 업무 시간이 부족함
- 팀 간 협업할 때 커뮤니케이션 채널이 불명확함
- 성과 평가 기준을 더 투명하게 공유해줬으면 좋겠음
...`}
</PromptBlock>

## 실습 2: 인사이트 보고서 초안

<PromptBlock title="VoE 인사이트 보고서 프롬프트">
{`위 분석 결과를 바탕으로 경영진 보고용 VoE 인사이트 보고서를 작성해줘.

[형식]
1. 요약 (3줄 이내)
2. 주요 발견사항 (TOP 3, 각 항목에 대표 의견 인용)
3. 우선순위별 개선 제안 (단기/중기/장기)
4. 다음 단계 (후속 조치 계획)

- 분량: A4 1페이지
- 톤: 데이터 기반, 객관적으로`}
</PromptBlock>

<HighlightBox type="expert" title="VoE 분석의 핵심">
  의견의 **양**이 아니라 **패턴**을 보세요. 
  10명이 같은 이슈를 다른 말로 표현한다면, 그것이 진짜 문제입니다.
  Gemini에게 "반복되는 패턴이 있는지 찾아줘"라고 요청해보세요.
</HighlightBox>

<HighlightBox type="tip" title="다음 챕터 연결">
  분석된 VoE를 바탕으로 **구성원에게 피드백을 전달하는 방법**은 다음 챕터 '말하기'에서 다룹니다.
</HighlightBox>
