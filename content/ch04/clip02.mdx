# 인터뷰 내용 요약 & 인사이트 도출

인터뷰가 끝나면 녹취나 메모를 정리해야 합니다. NotebookLM과 Gemini를 활용해 핵심 인사이트를 빠르게 추출해봅니다.

## 방법 1: Gemini로 텍스트 요약

인터뷰 내용이 텍스트로 있다면 Gemini에 직접 붙여넣어 분석할 수 있습니다.

<PromptBlock title="인터뷰 요약 프롬프트">
{`너는 HR 리서처야. 아래 인터뷰 내용을 분석해줘.

[작업]
1. 인터뷰이가 가장 강조한 포인트 3가지
2. 감정적으로 강하게 표현된 부분 (불만/기대/희망)
3. 숨겨진 니즈: 표면적 요청 뒤에 있는 진짜 필요
4. 후속 인터뷰에서 더 파야 할 질문 2개

[인터뷰 내용]
(여기에 인터뷰 녹취 또는 메모를 붙여넣으세요)

예시:
"솔직히 회의가 너무 많아요. 회의가 없으면 일을 더 잘할 수 있을 것 같은데...
그냥 보고용 회의가 대부분이고 실제로 결정되는 게 없어요."
...`}
</PromptBlock>

## 방법 2: NotebookLM으로 다중 인터뷰 교차 분석

여러 명의 인터뷰 내용을 한 번에 분석하려면 NotebookLM이 효과적입니다.

<StepByStep
  steps={[
    {
      number: 1,
      title: "NotebookLM 노트북 생성",
      description: "notebooklm.google.com → 새 노트북 만들기. 노트북 이름: '구성원 인터뷰 분석'",
    },
    {
      number: 2,
      title: "인터뷰 내용 업로드",
      description: "각 인터뷰 파일을 소스로 업로드. PDF, Google Docs, 복사한 텍스트 모두 가능.",
    },
    {
      number: 3,
      title: "교차 분석 질문",
      description: "아래 질문들을 입력하여 여러 인터뷰를 동시에 분석합니다.",
    }
  ]}
/>

<PromptBlock title="NotebookLM 교차 분석 질문">
{`인터뷰이들이 공통적으로 언급한 주제는 무엇인가요?`}
</PromptBlock>

<PromptBlock title="인사이트 도출 질문">
{`인터뷰 결과를 바탕으로 JB가 집중해야 할 3가지 우선순위를 제안해줘.
각 우선순위에 해당 근거가 되는 인터뷰 내용도 함께 인용해줘.`}
</PromptBlock>

<HighlightBox type="expert" title="NotebookLM의 강점">
  출처 인용 기능 덕분에 "어떤 인터뷰이가 이 말을 했는지" 바로 확인할 수 있습니다.
  보고서 작성 시 **근거 있는 인사이트**를 제시할 수 있어 신뢰도가 높아집니다.
</HighlightBox>
