<DeepDiveSection title="왜 AI Studio인가" id="ai-studio" clipId="ch06/clip03">

일반 Gemini 앱에서는 매번 프롬프트에 "정중하게 써줘", "200자 내외로" 같은 제약을 반복해서 써야 합니다. AI Studio에서는 이를 **System Instruction**으로 한 번 정의하고, 이후 모든 대화에 자동 적용합니다. 게다가 **Temperature**, **Top-P** 같은 생성 파라미터를 직접 튜닝할 수 있어 "일관성 ↔ 창의성" 트레이드오프를 눈으로 보며 조절할 수 있습니다.

<HighlightBox type="expert" title="R&D 관점 한 줄 요약">
Temperature는 모델 출력층의 소프트맥스 분포 평활화 파라미터입니다. 낮으면 argmax 쏠림, 높으면 균등에 가까움. "경영진 보고 = 낮게(0.2~0.4)" 는 여기서 나옵니다.
</HighlightBox>

## 실습 3: 멀티모달 — 차트 이미지 해석

<PromptBlock title="멀티모달 해석 요청">
{`업로드한 차트를 경영진 입장에서 해석해줘.

2. 경영진이 즉시 떠올릴 만한 반론 2개
3. 슬라이드 제목으로 쓸 수 있는 문구 3안 (각 20자 이내)`}

<HighlightBox type="expert" title="왜 이게 강력한가">
Gemini Pro는 차트의 시각 패턴을 "상승/하락/변곡점" 수준에서 읽어냅니다. 숫자가 적힌 slide의 경우 OCR로 숫자 추출까지 자동으로 합니다. 사람이 먼저 해석하기 전에 "AI가 먼저 본 인상"을 수집해 사각지대를 줄입니다.

## 실습 4: Function Calling 맛보기 (Agent 층 문턱)

Function Calling은 "모델이 스스로 필요한 도구를 판단해 호출"하는 기능입니다. 본격 구현은 안 하지만, **개념을 체험**해봅니다.

<PromptBlock title="Function 정의 (복붙)">
{`{
  "name": "get_department_headcount",
  "description": "특정 부서의 현재 구성원 수를 반환",
  "parameters": {
    "type": "object",
    "properties": {
      "department": {
        "type": "string",
        "description": "부서명 (예: R&D센터 소재기술)"

    "required": ["department"]

## Slide 초안 생성 프롬프트

튜닝된 Gem을 이용해 슬라이드 초안을 뽑습니다.

<PromptBlock title="보고서 → 슬라이드 변환">
{`이 Gem으로 변환한 경영진 보고문을 Google Slides 10장 구성의 슬라이드 초안으로 바꿔줘.

[각 슬라이드에 포함]
- 제목 (10자 이내)
- 본문 키포인트 (3개 이내, 각 15자 이내)
- 발표자 노트 (말할 내용 2~3문장)
- 권장 시각 자료 (차트 유형 또는 아이콘 제안)

[슬라이드 구성]
1. 표지 / 2. Executive Summary / 3. 현황 / 4. 근거 데이터 / 5. 원인 분석
6. 대안 1 / 7. 대안 2 / 8. 대안 3 / 9. 기대효과 / 10. 요청사항`}
</PromptBlock>

<HighlightBox type="expert" title="강사 팁">
슬라이드에서 가장 중요한 건 **"3초 보고 핵심을 알 수 있는가"**. 튜닝된 Gem은 매번 이 규칙을 자동 적용합니다. **프롬프트를 외우는 것이 아니라 Gem을 저장하는 것 — 이게 반복 업무의 진짜 자동화입니다.**
</HighlightBox>

<HighlightBox type="tip" title="내 업무에 적용하기">
- 본인 팀 컨텍스트(조직·용어·톤)를 반영한 "○○팀 전용 Pitch Gem" 1개 제작 → 다음 경영진 보고부터 즉시 사용.
- Function Calling은 오늘은 맛만 보지만, 사내 IT와 연결되면 "구성원 수 자동 조회 → 보고문 자동 포함"까지 확장됩니다.
</HighlightBox>
