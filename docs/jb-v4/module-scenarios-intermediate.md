# 나머지 3개 모듈 중급 실습 시나리오 (v4)

> Deep Dive 3종(듣기·설득하기·축적하기)과 같은 포맷. 시간 배분은 4시간 과정 중 이 3개 합산 65분.

---

## M2. 말하기(Caring Message) — 관계 해치지 않는 피드백 전달

**배치**: 본 모듈 후반 (소요 25분) · **산출물**: 피드백 메일 초안 3종 + Caring Message Gem

### 딥다이브 3축
1. **페르소나 프롬프팅** — 수신자의 성향·직급·과거 반응을 모델에 주입하면 "톤"이 자동 재구성됨
2. **톤 레지스터 전환 매트릭스** — 톤은 단어가 아닌 **문장 구조와 정보 순서**에서 나옴. 정중형/단호형/공감형 3축 비교
3. **수신자 시뮬레이션 검증 루프** — 작성 후 "이 메일을 받은 사람의 예상 반응 5줄"을 역생성해 톤 실패 자기검열

### 사전 준비 (강사)
- 원본 피드백 상황 시나리오 1건 (복붙용): *"신입 연구원이 주간보고 작성을 3주째 누락. 선임이 불편함을 표출한 상황에서 본인이 중재 메일을 쓰는 JB 역할"*
- 3인 수신자 페르소나 템플릿 (복붙용): A. 30대 팀장 (직설적·성과 지향), B. 50대 선임 (관계 중시·체면 민감), C. 20대 신입 (피드백에 과민)
- 톤 매트릭스 3종 복붙 프리셋

### Step-by-Step
| Step | 시간 | 활동 | 딥다이브 포인트 |
|:---:|:---:|---|---|
| 앵커 | 3분 | 톤은 단어가 아닌 문장 구조·정보 순서에서 나옴 (Before/After 1쌍 시연) | 원리 |
| 1 | 5분 | 원본 상황 입력 → 기본 프롬프트로 메일 생성 (bland 버전 확보) | 베이스라인 |
| 2 | 7분 | 수신자 페르소나 3종 주입 → 같은 내용을 3개 버전으로 재생성 → 차이 관찰 | 페르소나 효과 |
| 3 | 5분 | 3가지 톤(정중/단호/공감)으로 재요청 → 언제 어느 톤이 맞는지 토의 | 톤 레지스터 |
| 4 | 3분 | 완성 메일을 다시 Gemini에 넣고 *"이 메일을 받은 사람의 예상 반응 5줄"* 역생성 → 실패 지점 교정 | 검증 루프 |
| 5 | 2분 | 최종 설정을 "Caring Message Gem"으로 저장 | 자산화 |

### 완료 체크리스트
- [ ] 페르소나별 메일 3종 확보
- [ ] 톤 3종 비교 경험
- [ ] 수신자 시뮬레이션 역검증 1회 수행
- [ ] Caring Message Gem 저장 완료

### 월요일 활용
실제 피드백 메일 작성 전 Gem에 초안 넣어 **톤 보정 + 수신자 반응 시뮬레이션**을 루틴으로 도입.

## M3. 질문하기(Persona Kit) — 현장 구성원 인터뷰 설계

**배치**: 본 모듈 전반 (소요 20분) · **산출물**: 페르소나 프로파일 1건 + 4단 질문지 + Persona Kit Gem

### 딥다이브 3축
1. **페르소나 합성** — 단편 정보(직급·나이·부서) → 입체적 인물상(관심사·스트레스·커뮤니케이션 선호)으로 확장하는 프롬프팅
2. **Deep Research 기반 맥락 보강** — 해당 직군·업계 최근 이슈를 페르소나에 주입해 현실성 상승
3. **4단 질문 계위 설계** — 사실확인 → 경험 → 해석 → 미래. R&D 연구자에게 익숙한 "가설→검증" 구조와 유사

### 사전 준비 (강사)
- 가상 인터뷰 대상자 기초 정보 (복붙용): *"52세, R&D센터 선임연구원, 20년 경력, 최근 팀 개편 후 리더 역할 이관 중"*
- Deep Research 시드 프롬프트 (복붙용): *"국내 R&D 조직에서 50대 선임이 리더 이관기에 겪는 공통 이슈 5개"*
- 4단 질문 계위 템플릿

### Step-by-Step
| Step | 시간 | 활동 | 딥다이브 포인트 |
|:---:|:---:|---|---|
| 앵커 | 3분 | 좋은 인터뷰 = 상대의 사고 경로를 따라가는 것. 4단 계위 소개 | 원리 |
| 1 | 4분 | 기초 정보 입력 → Gemini로 페르소나 프로파일 생성 (관심사·스트레스·말투) | 페르소나 합성 |
| 2 | 6분 | Deep Research 실행 → 결과를 페르소나에 주입 *"위 리서치를 반영해 이 사람의 현재 가장 민감한 주제 3개 추출"* | 맥락 보강 |
| 3 | 5분 | 4단 계위 질문지 생성 (사실·경험·해석·미래 각 2문항) → 실제 이 사람에게 할 수 있는 수준인지 점검 | 질문 계위 |
| 4 | 2분 | 페르소나+질문 생성 워크플로우를 "Persona Kit Gem"으로 저장 | 자산화 |

### 완료 체크리스트
- [ ] 입체적 페르소나 1건 확보
- [ ] Deep Research 결과가 질문에 반영됐는지 역추적 가능
- [ ] 4단 계위 질문지 8문항 확보
- [ ] Persona Kit Gem 저장 완료

### 월요일 활용
다음 인터뷰 전 **대상자 기초 정보 3줄만 입력하면 10분 내 맞춤 질문지**가 나오는 루틴 확보.

## M4. 실행하기(Action Plan) — 간담회 결과를 실행으로 연결

**배치**: 본 모듈 전체 (소요 20분) · **산출물**: 액션플랜 문서 1건 + 우선순위 매트릭스 차트 + Function 호출 로그
**Sheets 제거**: CSV + Gemini 코드 생성 + Canvas 경로로 대체

### 딥다이브 3축
1. **"이슈 → 액션 아이템" 4요소 변환** — 주어·동사·마감·담당이 모두 명시된 문장만이 실행 가능. R&D의 실험 설계(objective·method·deadline·owner)와 동형
2. **임팩트×난이도 매트릭스 차트 코드 생성** — Gemini가 Matplotlib 코드를 생성 → Colab에서 즉시 실행. Sheets 없이도 시각화
3. **AI Studio Function Calling 맛보기** — `suggest_owner(task, dept)` 1개 함수 정의 → 모델이 스스로 호출 시도. Agent 층 문턱 체험

### 사전 준비 (강사)
- 이슈 CSV 샘플 10건 (복붙용) — 실습 1(듣기)에서 도출한 VoE 이슈 확장
- Colab 링크 1개 (Python 환경 없이 코드 즉시 실행)
- Function Calling JSON 템플릿 (복붙용)

### Step-by-Step
| Step | 시간 | 활동 | 딥다이브 포인트 |
|:---:|:---:|---|---|
| 앵커 | 3분 | 실행 가능한 과제의 4요소 (주어·동사·마감·담당) + R&D 실험 설계와의 동형성 | 원리 |
| 1 | 5분 | M1의 VoE 이슈 3개 입력 → Gemini에 *"4요소 액션 아이템으로 분해"* | 변환 패턴 |
| 2 | 6분 | 이슈 10건 CSV 업로드 → *"임팩트(1-5)·난이도(1-5) 평가 후 Matplotlib 매트릭스 차트 코드 생성"* → Colab에서 실행 → PNG 저장 | 시각화 자동화 |
| 3 | 4분 | AI Studio 접속 → Function Calling 템플릿 붙여넣기 → *"이 액션 아이템 담당자를 제안하려면 어떤 함수가 필요한가?"* → 모델이 함수 호출 시도 관찰 | Agent 체험 |
| 4 | 2분 | 최종 액션플랜을 Canvas에 붙여넣고 1페이지 문서화 | 산출물 포맷 |

### 완료 체크리스트
- [ ] 4요소 액션 아이템 3건 확보
- [ ] 임팩트×난이도 매트릭스 차트 이미지 생성 완료
- [ ] Function Calling 호출 로그 1회 확인
- [ ] Canvas 액션플랜 문서 1페이지 완성

### 월요일 활용
간담회 직후 이슈 CSV만 준비하면 **매트릭스 차트 자동 생성** + 액션플랜 문서화까지 30분 내 완료하는 루틴.
